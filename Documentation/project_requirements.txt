Users (faculty/coordinators) provide details across the following report sections:
● General points : department meetings, parent-teacher meetings, and institutional
announcements
● Faculty joined / relieved : designation, joining or relieving date
● Faculty achievements : awards, guest lecture invitations, reviewer/jury roles
● Student achievements : individual recognitions with roll number and department
● Department achievements : collective recognitions or milestones
● Faculty events conducted : FDPs, workshops, STTPs with resource persons and
participation count
● Student events conducted : technical workshops, guest lectures, topic and coordinator
details
● Non-technical events conducted : cultural, social, and institutional events (tracked
separately from technical events)
● Industry/college visits : institution name, location, coordinator, student count, and dates
● Hackathon / external event participation : event name, organizer, mentor, and student
count
● Faculty FDP / certification attendance : NPTEL, Coursera, EDX, industry certifications
● Faculty visits: faculty members visiting other colleges or industry locations
● Patents published: title, application number, and publication date
● VEDIC programs: institution-specific training programs for students and faculty
(Hyderabad / Bangalore centres)
● Placements: company name, department, number of students placed, and package
details
● MoUs signed: organization name, signing date, validity period, and purpose
● Skill development programs: domain-specific training topics, coordinators, sessions,
and student counts
The system should:
● Allow multiple faculty members and coordinators to contribute data collaboratively in
real time
● Organize all information across the 17 predefined report sections in a structured
manner
● Support editing, updating, and deletion of individual entries within each section
● Generate a complete weekly report in the institution's predefined format automatically
● Provide a dashboard for quick validation and review before report submission
● Support dynamic addition of multiple entries (rows) within each section to handle
multiple records for the same reporting period
● Provide a live preview of the final report in the institutional format before export and
submission
Requirements
● Role-based login for faculty, coordinators, and admin with appropriate access controls
● Section-wise structured data entry forms for all 17 report components with field-level
validation
● Shared, team-based contribution- multiple users can fill different sections
simultaneously
● Track and associate each entry with the contributing faculty member for accountability
and traceability
● Edit and update options for all submitted entries within the active reporting week
● Centralized database storage with week-based report isolation
● Ensure all entries are validated against the selected weekly duration so that only
relevant data is included in the report
● Automatic weekly report generation in the institution's structured format (exportable as
PDF/DOCX)
● Dashboard for coordinators and admins to validate, review, and track completion status
of all sections
● Section completion indicators showing which sections are pending, in progress, or
complete
